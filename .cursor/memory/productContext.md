# パフォーマンスチケット販売プラットフォーム「Dance」

このドキュメントは、.cursor/flow/developmentFlow.md に基づくプロダクト概要の理解・要件のヒアリングフェーズの成果物です。

## プロダクト概要

「Dance」は、Peatix を参考にした公演に特化したチケット販売プラットフォームです。イベント情報の集積と電子チケット販売を実現する Web プラットフォームとして構築されます。特に舞台、コンサート、ダンスショーなどのパフォーマンス系イベントに焦点を当て、主催者とイベント参加者の双方にシームレスな体験を提供します。

## ターゲットユーザー

### 1. イベント主催者

- プロフェッショナルなイベント企画会社
- 文化団体、アート関連団体
- 個人のパフォーマー、アーティスト
- 学校や教育機関（発表会、演奏会など）

### 2. イベント参加者（チケット購入者）

- 文化・芸術愛好家
- イベント参加を楽しむ一般消費者
- 特定のアーティスト/パフォーマーのファン
- ファミリー層（子供の発表会など）

### 3. イベント運営スタッフ

- 会場の入場管理担当者
- イベント当日の運営スタッフ

## 主要ユースケース

### イベント主催者

1. イベント情報の登録・管理
   - イベント詳細情報の入力
   - 画像や関連ファイルのアップロード
   - 開催日時・場所の設定
2. チケット販売管理
   - チケット種別・価格の設定
   - 座席情報の登録・管理
   - 販売期間の設定
3. 売上・参加者管理
   - 販売状況のリアルタイム確認
   - 売上レポートの閲覧・ダウンロード
   - 参加者情報の管理

### イベント参加者

1. イベント検索・閲覧
   - イベントの検索・一覧表示
   - イベント詳細情報の閲覧
   - イベントカレンダーの利用
2. チケット購入
   - 座席選択
   - 決済処理
   - 電子チケット（QR コード）の取得
3. チケット管理
   - 購入履歴の確認
   - QR コードの表示・保存
   - イベント情報の更新通知受信

### イベント運営スタッフ

1. 入場管理
   - QR コード読取による入場確認
   - 入場記録の管理
   - リアルタイム入場者数の把握
2. 運営支援
   - スタッフ間の情報共有
   - トラブル対応の記録

## MVP の主要システム

システムは以下の 3 つの主要コンポーネントから構成されます：

### 1. イベント管理システム

主催者向けのバックオフィス機能を提供し、イベント情報の登録・管理、チケット販売設定、売上レポートなどの機能を実装します。

### 2. チケット販売サイト

一般ユーザー向けのフロントエンドサイトで、イベント検索・閲覧、チケット購入、電子チケット管理などの機能を提供します。

### 3. チケット確認アプリ

イベント当日の入場管理を効率化するためのアプリケーションで、QR コード読取、入場記録管理、リアルタイム入場者数カウントなどの機能を実装します。

## プロダクトの独自の価値提案

1. **パフォーマンス特化型プラットフォーム**

   - 舞台、コンサート、ダンスショーなどのパフォーマンス系イベントに最適化された機能
   - 座席選択システムによる会場レイアウトの視覚的表現

2. **効率的なチケット管理**

   - QR コードベースの電子チケットによるペーパーレス化
   - スマートフォン表示に最適化されたチケット管理

3. **シームレスな運営管理**

   - リアルタイムの入場管理とデータ収集
   - 簡易的なスタッフモードによる運営支援

4. **アクセシビリティと使いやすさ**
   - 直感的なユーザーインターフェイス
   - アカウント登録任意のチケット購入フロー
   - イベントカレンダー連携による情報アクセス性向上

## 技術要件とプラットフォーム

- **フロントエンド**: Next.js 14（App Router）
- **スタイリング**: Tailwind CSS
- **バックエンド**: Supabase（PostgreSQL、認証、ストレージ）
- **開発言語**: TypeScript
- **デプロイ**: Vercel または類似のサービス
- **その他の技術**: QR コード生成・読取、決済処理 API 連携

## ビジネスモデル

1. **手数料モデル**

   - チケット販売価格に対する一定割合の手数料（5〜10%を想定）
   - 決済処理手数料（別途）

2. **オプション機能の有料提供**
   - 高度な分析レポート
   - カスタマイズされたイベントページ
   - 優先サポート

## 法規制と考慮事項

- 個人情報保護法に準拠したユーザーデータの取り扱い
- 決済セキュリティに関する規制への準拠
- 電子チケットの法的有効性の確保

## 初期ローンチの範囲（MVP）

MVP として最小限の機能セットでローンチし、ユーザーフィードバックを基に段階的に機能を拡張していく方針です。初期ローンチに含まれる機能は以下の通りです：

1. イベント情報登録・編集
2. 基本的なチケット販売機能
3. QR コードベースの電子チケット
4. シンプルな入場管理機能
